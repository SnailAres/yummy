# 微信开发者工具清除缓存指南

## 问题描述
```
error occurs: no such file or directory, access 'wxfile://usr/miniprogramLog/log2'
```

这个错误是微信开发者工具的缓存问题导致的，即使您使用的是 Windows 系统，开发者工具内部可能存在残留的 Unix 风格路径配置。

---

## 解决方案

### 方法一：清除开发者工具缓存（推荐）

#### 1. 清除项目缓存
- 在微信开发者工具中，点击菜单栏的「工具」按钮
- 选择「清除缓存」
- 勾选以下选项：
  - ☑ 文件缓存
  - ☑ 数据缓存
  - ☑ 授权数据
  - ☑ 安全域名校验信息
- 点击「确定」

#### 2. 清除工具缓存并重启
- 点击菜单栏的「工具」
- 选择「清除缓存」
- 选择「清除全部缓存」
- 彻底关闭微信开发者工具（不是最小化）
- 重新打开开发者工具

#### 3. 删除项目本地缓存文件
在 Windows 系统中，删除以下目录：
```
C:\Users\你的用户名\AppData\Local\微信开发者工具\User Data\Default\IndexedDB
```

**操作步骤**：
1. 按 `Win + R` 打开运行窗口
2. 输入：`%appdata%` 并回车
3. 找到 `Local` 文件夹
4. 进入 `微信开发者工具\User Data\Default`
5. 删除整个 `IndexedDB` 文件夹（如果存在）

---

### 方法二：重新导入项目

1. 在微信开发者工具中关闭当前项目
2. 点击「导入项目」
3. 重新选择项目目录：
   ```
   D:\Users\55011\WeChatProjects\miniprogram-1
   ```
4. 点击「导入」
5. 等待项目加载完成

---

### 方法三：修改项目配置

检查项目配置文件是否正确，确保 `miniprogramRoot` 指向正确的目录。

当前配置（已验证正确）：
```json
{
  "miniprogramRoot": "miniprogram/",
  "cloudfunctionRoot": "cloudfunctions/"
}
```

---

### 方法四：使用命令行清除缓存

如果以上方法无效，可以尝试使用命令行清除缓存：

```powershell
# 在 PowerShell 中执行
Get-ChildItem -Path "C:\Users\$env:USERNAME\AppData\Local\微信开发者工具\User Data\Default" -Recurse -Include "log*", "Cache*", "IndexedDB" | Remove-Item -Recurse -Force
```

## 预防措施

### 1. 定期清理
建议定期（每周或每月）执行一次缓存清理，保持开发者工具流畅运行。

### 2. 避免跨平台文件操作
- 不要在 WSL（Windows Subsystem for Linux）和 Windows 之间频繁复制项目文件
- 使用统一的开发环境

### 3. 更新开发者工具
确保使用最新版本的微信开发者工具：
- 点击菜单栏的「帮助」
- 选择「检查更新」
- 如有新版本，请及时更新

---

## 验证修复

清除缓存并重启后：
1. 重新打开项目
2. 点击「编译」按钮
3. 检查控制台是否还有 `wxfile://usr/miniprogramLog/log` 相关错误
4. 尝试测试云开发功能（点击「云开发」按钮）

---

## 如果问题仍然存在

如果以上方法都无法解决问题，请尝试：

1. **卸载并重新安装微信开发者工具**
2. **检查系统环境**：确保 Windows 系统没有 WSL 影响文件系统
3. **联系微信小程序官方客服**反馈此问题

## 常见问题

### Q: 为什么会出现这种 Unix 风格路径？
A: 这可能是因为项目文件曾经在其他系统（如 macOS 或 Linux）上编辑过，或者开发者工具的缓存管理模块存在 bug。

### Q: 清除缓存会删除我的代码吗？
A: 不会。清除缓存只删除临时文件、日志和编译产物，不会影响源代码。

### Q: 如果清除缓存后仍然报错怎么办？
A: 建议重新导入项目或联系开发商反馈此问题。

---

**提示**：请按照方法一和方法二的顺序执行，通常可以解决此问题。
